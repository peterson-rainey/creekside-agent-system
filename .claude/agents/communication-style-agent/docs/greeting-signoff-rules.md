# Greeting and Sign-off Rules

## Greeting Rules

| Context | Greeting | Frequency |
|---------|----------|-----------|
| Internal team (ongoing thread) | None -- jumps to content | 82% of all messages |
| Cade | None, or "Hey man" for new topic | Always |
| VAs/EAs | None, or @mention only | Always |
| New client (first email) | "Hi [First Name]," | Always |
| Existing client (reply) | None | Usually |
| Existing client (new thread) | "Hey [First Name]," | Usually |
| Warm lead (met in person/referred) | "Hey [First Name]," | Always |
| Cold lead (Upwork) | None -- opens with problem diagnosis | Always |
| Partner (familiar) | "Hey [First Name]," or none | Varies |
| Formal/legal | "Hi [First Name]," | Always |

**"Hey" vs "Hi" rule:** "Hey" = 81% of greetings (existing relationships, casual). "Hi" = 17% (new contacts, formal). "Hello" = 2% (extremely rare, most formal).

**He NEVER uses a greeting + pleasantry combo.** When he does greet, the next word is business content.

## Sign-off Rules

| Context | Sign-off | Frequency |
|---------|----------|-----------|
| Internal (all platforms) | None | Always |
| Client (ongoing thread) | None, or "let me know" | Usually |
| Formal call recap | "-- Peterson\nCreekside Marketing Pros" | Always |
| Follow-up to stalled lead | None, or "Let me know!" | Usually |
| New client onboarding template | None (ends with last instruction) | Always |
| Collections/legal | Business signature block | Always |

**96% of all messages have NO formal sign-off.** "Let me know" (475 instances) is the closest thing to a universal closer, but it's functional, not social.
