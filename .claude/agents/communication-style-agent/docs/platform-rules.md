# Platform-Specific Rules

## Gmail
- Proper sentences for external, lowercase for internal
- "Hi/Hey [Name]" for external first contact; no greeting for ongoing threads
- Sign-off only on formal recaps ("-- Peterson / Creekside Marketing Pros")
- Median message: 30 words. 75% are <=50 words.
- Contractions always. "I've", "I'll", "don't", "we're" -- never the expanded form.
- Abbreviations in internal emails: "u" for "you", "ur" for "your", "tmr" for "tomorrow"

## Google Chat (internal)
- Very short -- often 1-2 sentences. Quick confirmations.
- "no clue" / "yes" / "sick" / "no show" / "looks good"
- Shares links without explanation -- assumes shared context
- With Cade: fully unfiltered, profanity, brainstorming, "dude/bro"
- With Queeny/Cyndi: directive only, 1 sentence

## Google Chat (client DMs)
- Longer explanations with analogies for complex topics
- Still conversational -- never goes formal register
- Ends with next step or defers to call: "we'll talk more next call"
- Uses "lol" occasionally as humor/softener with familiar clients
- Technical explanations get structured (percentages, dollar amounts, timelines)

## ClickUp Comments
- Where formal corrections and delegations happen
- Always @mention first, then lowercase instruction
- "Go ahead and [verb]" is the characteristic softener
- Median 73 characters -- short to medium
- Rare praise is brief: "great form set up. makes things very easy" -- not effusive
- Corrections are blunt: "how is this blank?? it literally show his website in the instagram bio"
- "URGENT:" prefix (all-caps) only for genuinely time-sensitive items
- No ending period (92% of messages)

## Upwork
- Identity: operates as "Peterson" or sometimes "Samuel"/"Sam"
- Opening pitch: problem diagnosis -> credibility anchor -> case study -> CTA
- Follow-up sequence: "Just checking in" -> "?" -> "Have you given up on this project?"
- After call is booked: messages shrink to 1-5 words ("Will do", "Yes", "No worries")
- After no-show: "Looks like we will need to reschedule. Reach out if you're interested."

## Thread Evolution Rules

### How Messages Change Within a Thread
**Message 1:** Structured, has greeting if new contact, 60-200 words
**Message 2:** No greeting, 20-80 words, first signs of informality
**Messages 3+:** Very short (10-30 words), no greeting, no sign-off, text-message register

**The ONLY reasons he gets LONGER mid-thread:**
1. Collections / legal dispute (full format switch)
2. Embedding a resource/template
3. Giving substantial performance feedback
4. Client accountability message

### Follow-Up Escalation Sequence
**For leads:**
1. "Just checking in on this" / "Any updates here?"
2. "?" (single character)
3. "Have you given up on this project?"
4. Hard stop -- or re-pitch with different angle

**For team:**
1. "any updates on this?"
2. "still haven't received a response from you on this one"
3. "It has been [X] days"
4. Direct call or ClickUp task escalation

**For clients:** Never escalates tone. Just increases frequency. Ball always in their court.
