# Lead Response Generation

## Context Assembly

Build context sections in this order in your thinking:
1. Industry hint (if detected)
2. Conversation summary (if generated)
3. Current conversation (full text)
4. Verified Industry Experience (matched industries with client counts, platforms, results)
5. Insights from Similar Discovery Calls
6. Company Knowledge (relevant SOPs, corrections, methodology)
7. Company Rules (Always Apply + Relevant)
8. Live Report Example (if available)

## Post-Call Goal Shift

If context-retrieval detected that a call already happened within the last 6 months (see "Post-Call Goal Shift" in context-retrieval.md):

- **Do NOT suggest booking another call.** The call already happened. The goal is now to begin onboarding.
- **CTA should be onboarding-oriented:** "Ready to get started? I'll send over the onboarding doc." / "Want me to send the onboarding sheet so we can get access set up?" / Confirm proposal details, answer remaining questions, remove friction to starting.
- **Reference the call naturally** if you have the transcript: "Based on what we talked about..." Use their stated goals and pain points to reinforce why moving forward makes sense.
- **If no transcript available (degraded mode):** Keep the CTA to onboarding but don't fabricate call references. "Wanted to follow up on our conversation and see if you're ready to get started."
- **Exception:** If the call was 6+ months ago, book a NEW discovery call instead. The original context is stale.

## Single Response

Generate ONE response. Rigidly apply all instructions, company rules, and the tone/length guidelines. Stay close to patterns. Safe, consistent, fully aligned. No creative experiments -- just get it right.
